# ALUM(cm) — חבילת Beta להעלאה

## מה יש בחבילה

```
alumcm-beta/
├── index.html       ← האפליקציה המלאה (קובץ עצמאי)
├── README.md        ← הקובץ הזה
├── .htaccess        ← הגדרות Apache (אופציונלי)
└── manifest.json    ← הגדרות PWA למובייל
```

## איך מעלים לאוויר

### אופציה 1: שרת מארח רגיל (cPanel / FTP)
1. העלה את **כל הקבצים** בחבילה הזו לשורש הדומיין
2. ודא שהקובץ נקרא `index.html`
3. גש לדומיין שלך — האפליקציה תעלה מיד

### אופציה 2: Netlify / Vercel (חינם, מומלץ)
1. גרור את התיקייה כולה ל-Netlify Drop: https://app.netlify.com/drop
2. תקבל URL זמני תוך 30 שניות
3. אופציונלי: חבר את הדומיין שלך מהגדרות Netlify

### אופציה 3: GitHub Pages (חינם)
1. צור Repository חדש ב-GitHub
2. העלה את הקבצים
3. הפעל GitHub Pages מ-Settings → Pages
4. תקבל URL ב-`username.github.io/repo-name`

## ניהול המנויים בחבילה הזו

חבילה זו היא **גרסת Beta עצמאית** — כל הנתונים נשמרים מקומית
ב-IndexedDB של הדפדפן של כל משתמש. אין שרת מרכזי.

### איך עובד תהליך התשלום בגרסת ה-Beta:

1. משתמש לוחץ "שדרג ל-Pro"
2. רואה מודאל סימולציית PayPlus עם כרטיס לדוגמה
3. הופך ל-Pro מקומית (בדפדפן שלו בלבד)
4. רואה חשבונית מדומה, אימייל אישור מדומה

זה מצוין לבדיקת ה-UX וזרימת הפיצ'רים. **לא לחיובים אמיתיים.**

### למעבר לפרודקשן אמיתי:
ראה את `ALUMcm_Setup_Guide.docx` — מדריך מלא למעבר ל-Supabase + Grow + Invoice4U.

## שאלות תיקוף לבעלי מקצוע

הצעת שאלות שכדאי לשאול בכל בדיקה:

1. **ראשון רושם:** מה אתה חושב בעשר השניות הראשונות?
2. **כמה זמן לוקח** ליצור הצעת מחיר ראשונה?
3. **האם ברור** מה לעשות בכל מסך?
4. **מה היית משלם** עבור מערכת כזו? (₪50 / ₪99 / ₪150 לחודש?)
5. **אם הייתי מבטל את הפיצ'ר X — היית מתאכזב?**
6. **מה חסר** שעוצר אותך מלהשתמש?
